Payment questions allow respondents to make payments directly within a survey. Integrated with Stripe for secure payment processing, this element type supports configurable amounts, multiple currencies, and optional receipt handling — enabling use cases such as event registrations, donations, and product orders without leaving the survey experience.

<Note>The Payment question type is a planned feature for Typeform parity. This documentation describes the planned specification and will be updated upon implementation.</Note>

## Elements

### Title

Add a clear title to inform the respondent what payment is being requested (for example, "Complete Your Registration Payment" or "Make a Donation").

### Description

Provide an optional description with further details about the payment, such as what the payment covers, applicable terms, or refund policy information.

### Amount Configuration

Configure the payment amount as a fixed value. The amount is set during survey creation and cannot be changed by the respondent.

Amounts are specified as a positive integer in the smallest currency unit (for example, cents for USD). A value of `2999` represents $29.99.

### Currency

Select the currency for the payment. The following currencies are currently supported:

| Currency Code | Currency Name |
|---------------|---------------|
| `usd` | US Dollar |
| `eur` | Euro |
| `gbp` | British Pound |

The currency is specified as a lowercase ISO 4217 code. Additional currency support may be added in future releases.

### Stripe Integration

Payment processing is handled via Stripe. Each Payment element requires a `stripeIntegration` configuration object with the following fields:

- **`publicKey`** — The Stripe publishable key used for client-side Stripe Elements rendering
- **`priceId`** — The Stripe Price ID referencing the product/price in your Stripe account

<Note>Stripe credentials are configured per Payment element in the `stripeIntegration` object. The publishable key is the only key exposed to the client; server-side secret keys are used only in server actions for Payment Intent creation.</Note>

## Configuration

The following configuration options are available for Payment questions:

| Option | Description | Default |
|--------|-------------|---------|
| **Amount** | Payment amount as a positive integer in smallest currency unit (e.g., cents) | — |
| **Currency** | ISO 4217 currency code (`usd`, `eur`, `gbp`) | — |
| **Button label** | Custom label for the payment button (i18n supported, optional) | — |
| **Required** | Whether the payment must be completed to proceed | `true` |
| **Stripe integration** | Object containing `publicKey` (Stripe publishable key) and `priceId` (Stripe Price ID) | — |

Payment questions are always terminal within their block — they cannot have follow-up questions within the same block. This ensures the payment flow is completed before the respondent proceeds to the next section of the survey.

## Response Format

When a respondent completes a payment, the response data includes the Stripe payment intent details:

```json
{
  "paymentId": "pi_3N1234567890",
  "amount": 2999,
  "currency": "usd",
  "status": "succeeded",
  "receiptEmail": "respondent@example.com"
}
```

| Field | Type | Description |
|-------|------|-------------|
| `paymentId` | `string` | Stripe Payment Intent identifier |
| `amount` | `number` | Amount charged in smallest currency unit |
| `currency` | `string` | ISO 4217 currency code |
| `status` | `string` | Payment status (`succeeded`, `pending`, `failed`) |
| `receiptEmail` | `string` | Respondent email for receipt (if collected) |

## Planned Schema Specification

The Payment element type will extend the existing `ZSurveyElementBase` schema defined in `packages/types/surveys/elements.ts`. The following Zod schema represents the planned specification:

```typescript
// Extension to packages/types/surveys/elements.ts
export const ZSurveyPaymentElement = ZSurveyElementBase.extend({
  type: z.literal("payment"),
  currency: z.enum(["usd", "eur", "gbp"]),
  amount: z.number().int().positive().min(1),
  buttonLabel: ZI18nString.optional(),
  stripeIntegration: z.object({
    publicKey: z.string(),
    priceId: z.string(),
  }),
});

export type TSurveyPaymentElement = z.infer<typeof ZSurveyPaymentElement>;
```

The `Payment` entry has been added to `TSurveyElementTypeEnum` in `packages/types/surveys/constants.ts` as `Payment = "payment"`, extending the enum from 15 to 17 element types (together with `OpinionScale`).

<Note>Source reference: `packages/types/surveys/elements.ts`. The `currency` field uses a strict enum of three supported currencies. The `amount` field is a positive integer in the smallest currency unit (e.g., cents). The `stripeIntegration` object requires both a Stripe publishable key and a Stripe Price ID.</Note>

## API Request Example

The following JSON shows how a Payment question appears in a survey creation API payload:

```json
{
  "id": "payment_q1",
  "type": "payment",
  "headline": { "default": "Complete your payment" },
  "subheader": { "default": "Secure payment processed via Stripe" },
  "required": true,
  "currency": "usd",
  "amount": 2999,
  "buttonLabel": { "default": "Pay now" },
  "stripeIntegration": {
    "publicKey": "pk_live_...",
    "priceId": "price_..."
  }
}
```

This payload structure follows the conventions established by other element types in the Formbricks survey schema. The `amount` is a positive integer in the smallest currency unit (e.g., `2999` = $29.99 USD). The `buttonLabel` uses the i18n object format. For the complete API reference, see the [API v1 Reference](/api-reference) or [API v2 Reference](/api-v2-reference).
