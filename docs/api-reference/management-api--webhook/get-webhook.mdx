openapi: get /api/v1/webhooks/{webhookId}
---

## Path Parameter

| Parameter   | Type   | Required | Description                                                                                       |
| ----------- | ------ | -------- | ------------------------------------------------------------------------------------------------- |
| `webhookId` | string | Yes      | The unique identifier (CUID2 format) of the webhook to retrieve.                                  |

The `webhookId` is returned when creating a webhook via [Create Webhook](/api-reference/management-api--webhook/create-webhook) or when listing all webhooks via [List Webhooks](/api-reference/management-api--webhook/get-all-webhooks). The API key used for the request must have `GET` permission for the webhook's associated environment.

## Response Structure

A successful request returns the webhook object wrapped in a `data` envelope. The response contains the following fields:

| Field           | Type       | Description                                                                                                |
| --------------- | ---------- | ---------------------------------------------------------------------------------------------------------- |
| `id`            | string     | The unique identifier (CUID2 format) of the webhook.                                                       |
| `name`          | string \| null | An optional display name for the webhook. May be `null` if no name was provided during creation.         |
| `createdAt`     | string     | ISO 8601 timestamp of when the webhook was created.                                                        |
| `updatedAt`     | string     | ISO 8601 timestamp of when the webhook was last updated.                                                   |
| `url`           | string     | The destination URL that receives webhook payloads.                                                        |
| `source`        | string     | The origin of the webhook. One of `user`, `zapier`, `make`, or `n8n`.                                      |
| `environmentId` | string     | The CUID2 identifier of the environment this webhook belongs to.                                           |
| `triggers`      | string[]   | An array of event triggers. Possible values: `responseCreated`, `responseUpdated`, `responseFinished`.      |
| `surveyIds`     | string[]   | An array of survey IDs (CUID2 format) the webhook is scoped to. An empty array means all surveys.          |

## Error Responses

The following error scenarios may be returned by this endpoint:

| Status Code | Error         | Description                                                                                           |
| ----------- | ------------- | ----------------------------------------------------------------------------------------------------- |
| `404`       | Not Found     | No webhook exists with the given `webhookId`.                                                         |
| `401`       | Unauthorized  | The API key does not have `GET` permission for the environment associated with the webhook.            |

Source: `apps/web/app/api/v1/webhooks/[webhookId]/route.ts`

<Note>
  If the webhook exists but belongs to a different environment than the one your API key has access to, the endpoint returns a **401 Unauthorized** response rather than a **404 Not Found**. Ensure your API key is scoped to the correct environment.
</Note>
